<GuideSection id="skins" title="Skins over one component tree">
  Every skin owns the same three files over one component source. Its `theme.css` explicitly resolves the complete token
  contract for light and dark modes; it never inherits another skin's values. `skin.css` handles CSS that tokens and classes
  cannot express, and `skin.config.tsx` supplies typed design-system choices, slot overrides, and optional adornments.

<GuideSection id="anatomy" title="Stable anatomy without runtime metadata">
  Every public part emits the same two attributes. `data-control-ui` marks Control UI ownership and names the component
  scope; `data-slot` names a part local to that scope. `data-skin` belongs on the skin boundary, including portal
  positioners. It is not repeated on every component.

```css
/* One exact part in one skin. */
[data-skin="rig"] :where([data-control-ui="chat-composer"][data-slot="root"]) { ... }

/* Every interactive Control UI root, without touching host application controls. */
[data-skin="rig"] [data-control-ui][data-control="true"] { ... }
```

  The generated `skin-contract.json` is the equivalent of a `defineAnatomy` catalog, but it is built from the component
  source instead of becoming a browser-side object. It lists scopes, parts, states, paints, adornments, semantic families,
  and registry ownership in one agent-readable artifact. Item API responses separate `ownScopes` from `installedScopes`,
  so an agent sees both the requested component and the anatomy brought by its dependency closure.
</GuideSection>

<GuideSection id="skin-cost" title="Keep the active skin sparse">
  `skin.config.tsx` is one shared ES module. Components do not reload or clone it: `skinSlot(scope, part, context)` performs
  one optional property lookup against the active object. A large config does remain in the active JavaScript bundle because
  dynamic slot lookup cannot tree-shake individual keys, so broad visual decisions belong in tokens and scoped CSS. Keep
  slots for exceptions that need context, paints for exclusive render treatments, and adornments for optional structure.
  Edit the installed component directly when the markup or behavior itself must change.

  Bundle tests cap every shipped config at 4 kB gzip. Refined and Flat remain nearly empty; richer packs pay only for their
  one active config, not once per rendered component.
</GuideSection>

  to undo.
