<GuideSection id="choose" title="Choose what to install">
  Choose one skin for the visual contract, then install one component when you want to compose the interface yourself or a
  block when you want a complete recipe. A skin is required: core owns the bindings and mechanics, while the selected skin
  owns every token value. The integration selector changes the provider-specific usage composition only; the installed UI
  accepts plain props and children from any runtime.

  For a new project, each skin page exposes an `all-<skin>` manifest that installs the same canonical component set plus the
  chosen skin in one command. Changing the selected skin changes only skin-owned files; component source stays invariant.

</GuideSection>

<GuideSection id="install-skin" title="Install a skin">
  Start with Refined or choose another pack from the Skins catalog. Exactly one active pack owns `skin.config.tsx`,
  `skin-theme.css`, and `skin.css`; installing another pack replaces those three files.

  The current skin anatomy contract is version 5. Version 5 adds generated `data-popup-part` membership and resolves shared
  popup family classes before exact component slots. Reinstall core, popup components or their containing blocks, and the
  selected skin together so source, generated anatomy, and skin configuration remain in sync.

  Version 4 replaced ToolCall with Activity's typed tool variant. If upgrading from version 3 or earlier, remove both old
  install layouts before reinstalling:

```bash
rm -f components/control-ui/tool-call.tsx components/control-ui/hooks/use-tool-call.ts
rm -f src/components/control-ui/tool-call.tsx src/components/control-ui/hooks/use-tool-call.ts
```

  Replace `ToolCall` with `<Activity kind="tool" name="…">`, use the matching `Activity*` parts, replace input and output
  wrappers with `ActivityDetail`, `ActivityDetailLabel`, and `ActivityDetailContent`, and remove `useToolCall` in favor of
  mapping provider status directly to `ActivityState`. This pre-release contract intentionally ships no legacy selectors.

<GuideSection id="wire-css" title="Wire the CSS">
  The registry writes required CSS files beside the installed components, but it cannot add imports to your application.
  Add these imports once. Core `theme.css` exposes Tailwind bindings and invariant mechanics; the selected pack's
  `skin-theme.css` supplies the complete token contract and `skin.css` supplies its structural details.

```css
/* app/globals.css */
@import "tailwindcss";
@import "../components/control-ui/styles/theme.css";
@import "../components/control-ui/styles/effects.css";

@import "../components/control-ui/styles/skin-theme.css";
@import "../components/control-ui/styles/skin.css";
```

  Stamp the installed pack id on the root. There is no implicit token fallback when `data-skin` is missing or invalid.

```tsx
<html data-skin="refined">…</html>
```
</GuideSection>
